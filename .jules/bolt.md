## 2024-06-16 - [Optimizing Image Processing Pipeline]
**Learning:** Sequential processing of images using `await` in a loop is a major bottleneck in vanilla JS tools. Browsers can efficiently parallelize image decoding and canvas operations if triggered via `Promise.all`. Additionally, `FileReader.readAsDataURL` creates massive memory pressure via Base64 encoding; `URL.createObjectURL` is a zero-copy alternative that is significantly faster and more memory-efficient.
**Action:** Always prefer `URL.createObjectURL` for previews and result displays. Ensure every created Object URL is tracked and revoked using `URL.revokeObjectURL` in all cleanup paths (removal, clearing, or re-processing). Use `Promise.all` for batch asynchronous operations to utilize browser-level parallelism.
